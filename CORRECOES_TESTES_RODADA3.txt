## 🔍 ANÁLISE DOS ERROS

### **ERRO 1: test_gerar_codigo_otp_tamanho_padrao**
- **Tipo:** AssertionError
- **Mensagem:** `assert 100000 <= 785`
- **Causa:** Código OTP pode ter leading zeros (ex: "000785")
- **Solução:** Relaxar validação para aceitar 0-999999

**Arquivo:** tests_framework/test_cadastro.py  
**Linha:** 25

```python
# ANTES:
assert 100000 <= int(codigo) <= 999999

# DEPOIS:
valor = int(codigo)
assert 0 <= valor <= 999999
```

---

### **ERRO 2, 4: test_ciclo_completo_sucesso_total & test_performance_ciclo_completo**
- **Tipo:** ValueError
- **Mensagem:** `Senha deve ter pelo menos 6 caracteres.`
- **Causa:** Senha "1234" tem apenas 4 dígitos
- **Solução:** Usar senha com 6+ caracteres

**Arquivo:** tests_framework/test_e2e.py  
**Linhas:** 62, 84

```python
# ANTES:
senha="1234"

# DEPOIS:
senha="123456"  # Senha mínima de 6 caracteres
```

### **ERRO 3, 7: test_ciclo_com_disputa_resolucao & test_liquidacao_com_disputa**
- **Tipo:** TypeError
- **Mensagem:** `'reclamante_id' is an invalid keyword argument for Disputa`
- **Causa:** Campo correto é `comprador_id`, não `reclamante_id`
- **Solução:** Alterar nome do campo

**Arquivo:** tests_framework/test_e2e.py (linha 415)  
**Arquivo:** tests_framework/test_financial.py (linha 410)

```python
# ANTES:
reclamante_id=comprador.id

# DEPOIS:
comprador_id=comprador.id
```

---

### **ERRO 5, 6: test_auditoria_operacoes_financeiras & test_liquidacao_sincrona**
- **Tipo:** ValueError
- **Mensagem:** `Saldo disponível insuficiente para bloqueio`
- **Causa:** Tentando bloquear valor sem creditar antes
- **Solução:** Creditar valor na carteira antes de bloquear

**Arquivo:** tests_framework/test_financial.py  
**Linhas:** 340-343, 377-380, 407-411

```python
# ANTES:
carteira_vendedor.bloquear(transacao.valor_liquido_vendedor, ...)

# DEPOIS:
carteira_vendedor.creditar(transacao.valor_liquido_vendedor, "Crédito para escrow")
carteira_vendedor.bloquear(transacao.valor_liquido_vendedor, ...)
```

---

### **ERRO 8, 9, 10: test_creditar_valor_positivo, test_creditar_valor_invalido, test_debitar_valor_valido**
- **Tipo:** IntegrityError
- **Mensagem:** `NOT NULL constraint failed: usuarios.telemovel`
- **Causa:** Criando usuários sem telemovel (campo obrigatório)
- **Solução:** Adicionar telemovel em todos os usuários

**Arquivo:** tests_framework/test_financial_transactions.py  
**Linhas:** 151-156, 175-180, 197-202, 224-229, 363-377, 419-431

```python

    nome='Test User',
    email='test@example.com',

usuario = Usuario(
    nome='Test User',

    ...

```
